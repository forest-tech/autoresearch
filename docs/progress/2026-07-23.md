### log で`"module": executable file not found in $PATH` が出る

- ジョブスクリプト `train.sh` の中で、`module load` している(cuda, singularity)が、コメントアウトして確認した結果、どちらも問題ではない様に考えられる
- となると`train.sh` 内の`singularity exec` コマンドが怪しい
    - 利用しているsingularityイメージの`nlp-singularity.def`が [元のリポジトリ](http://github.com/cl-tohoku/nlp-singularity/tree/edfdca2811970bfb7d5aff7c9ba9452b365d4e8a) と異なっている可能性があるので、再度git clone してsingularityイメージを作り直す
    - ビルドエラー。/tmp が XFS の quota 管理下にあり、Singularity が一時ディレクトリへ展開する際に quota 制限に達したため、ビルドが失敗した。

````chat

原因を切り分けるには、まず一時ディレクトリを /tmp ではなくホームディレクトリなど容量に余裕のある場所へ変更してビルドします。

```sh
mkdir -p ~/tmp

export TMPDIR=$HOME/tmp
export SINGULARITY_TMPDIR=$HOME/tmp

singularity build --fakeroot nlp-singularity.sif nlp-singularity.def
```

これで成功すれば、原因は /tmp 側の quota または一時領域の制限だったことがほぼ確定します。

一方、この設定でも同じ disk quota exceeded が発生する場合は、HPC環境側の quota 設定や fakeroot の制限が原因である可能性が高く、システム管理者への確認が必要になります。
````
