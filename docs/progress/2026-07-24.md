### Singularity ビルドエラー①：`disk quota exceeded`

`singularity build --fakeroot` 実行時に、OCI イメージの展開中に以下のエラーが発生。

```text
write /tmp/build-temp-.../rootfs/usr/local/lib/libmkl_avx.so.1:
disk quota exceeded
```

#### 原因

- Singularity はビルド時にコンテナイメージを `/tmp/build-temp-*` に一時展開する。
- `/tmp` の空き容量は約226 GB、inodeも十分残っていたため、ディスク容量不足ではない。
- `/tmp` は XFS (`usrquota`, `grpquota`) で管理されており、quota により一時領域への書き込みが制限されたためビルドが失敗したと考えられる。

#### 確認した内容

```sh
df -h /tmp
quota -s
df -ih /tmp
mount | grep /tmp
singularity cache list
```

#### 対応

一時ディレクトリをホームディレクトリへ変更してビルドを実施。

```sh
mkdir -p ~/tmp

export TMPDIR=$HOME/tmp
export SINGULARITY_TMPDIR=$HOME/tmp

singularity build --fakeroot nlp-singularity.sif nlp-singularity.def
```

### Singularity ビルドエラー②：`Operation not permitted`

一時ディレクトリをホームディレクトリへ変更した後は、`disk quota exceeded` は解消されたが、`apt install` 実行中に以下のエラーが発生。

```text
dpkg: error processing archive .../libutempter0... (--unpack):
error setting ownership of './usr/lib/x86_64-linux-gnu/utempter/utempter':
Operation not permitted

E: Sub-process /usr/bin/dpkg returned an error code (1)
FATAL: While performing build: while running engine: exit status 100
```

#### 原因

- `apt install` の途中で、ファイルの所有者を変更する処理が実行された。
- しかし、現在のビルド環境ではその操作を行う権限がなく、`Operation not permitted` が発生した。
- このエラーにより `apt` が異常終了し、Singularity のビルドも失敗した。

#### 考えられる要因

- `fakeroot` 環境の制限
- 共有ファイルシステム上で `chown` が許可されていない
- `libutempter0` が特殊な権限設定を必要とする

- `libutempter0` がどのパッケージの依存で導入されているか調査する。
- 必要であれば、`tmux` や `libutempter0` をインストール対象から除外できるか検討する。
